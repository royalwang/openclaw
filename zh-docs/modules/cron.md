# 模块分析：Cron & Background Jobs

## 定时任务与调度 (Cron) - `src/cron/`

`cron` 模块构建了 OpenClaw 内部强大的异步任务与后台调度能力，确保系统能够高效地处理周期性维护任务和延迟消息投递。

### 核心组件

-   **任务服务 (`service.ts`)**:
    -   实现了完整的定时任务队列抽象。支持标准的 `cron` 表达式验证解析、一次性任务调度以及错失任务（Missed Jobs）补救。
    -   包含完善的防并发机制，避免同一任务在多实例或重启环节被多次抢占执行。
-   **持久化与迁移 (`store.ts`, `store-migration.ts`)**:
    -   基于 SQLite 高效存储任务状态（如参数序列化、下次运行时间、任务执行记录等）。
    -   提供完善的数据库迁移工具架构，确保版本平滑升级时定时任务数据不丢失。
-   **独立代理与投递 (`isolated-agent.ts`, `delivery.ts`)**:
    -   允许按计划在后台脱离用户交互直接唤醒 Agent（Isolated Agent），完成预设的后台研判任务后主动向设定渠道发起投递。
    -   支持复杂的投递策略逻辑，由于适配不同渠道的隔离特性，实现了如通知失败回滚（Failure Notify）、跨线程回复或论坛帖子（Forum Topics）的准确送达匹配机制。
-   **会话清理 (`session-reaper.ts`)**:
    -   负责周期性扫描并清理过期、非活跃的 Session 锁定与资源占用，保证系统长期运行下内存与临时存储的稳定性。
