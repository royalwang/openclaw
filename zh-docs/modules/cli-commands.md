# 模块分析：CLI & Commands

## 命令行界面 (CLI) - `src/cli/`

OpenClaw 的 CLI 设计非常注重启动性能和可扩展性。它没有采用一次性加载所有命令的传统方式，而是通过一套复杂的延迟加载机制来优化用户体验。

### 核心机制

-   **主入口 (`run-main.ts`)**:
    -   负责环境初始化（`.env` 加载）、Profile 处理（不同配置文件切换）以及运行时检查。
    -   在真正运行命令前，会根据 `argv.ts` 的预解析结果决定是否需要确保 `openclaw` 在系统 `PATH` 中。
-   **延迟加载与注册 (`command-registry.ts`)**:
    -   **占位符模式**: 初始仅注册命令的名称和描述作为“占位符”。
    -   **按需加载**: 只有当用户真正调用某个命令时，才会通过 `import()` 动态加载其完整实现模块。这使得即便拥有数百个命令，CLI 启动依然迅速。
-   **自定义参数解析 (`argv.ts`)**:
    -   在 Commander 介入前，先行解析“根选项”（如 `--profile`, `--verbose`）。
    -   提供细粒度的逻辑控制，例如决定哪些维护性任务（状态迁移、路径检查）可以针对特定命令跳过。

## 命令实现 (Commands) - `src/commands/`

该目录包含了所有内置功能的具体 CLI 命令实现。

### 组织结构

-   **按领域划分**: 命令通常按功能领域（如 `agents`, `config`, `gateway`, `channels`）组织成独立的文件或子目录。
-   **复杂命令拆分**: 对于复杂的命令组（如 `agents`），其实现被拆分为多个文件（例如 `agents.commands.add.ts`, `agents.commands.list.ts`），通过顶层文件（`agents.ts`）导出。
-   **交互式向导**: 许多关键命令（如 `onboard`, `configure`, `setup`）都与 `src/wizard/` 配合，提供友好的交互式引导。

### 设计优势

-   **高扩展性**: 插件系统可以利用相同的延迟加载注册机制，将其特有的 CLI 命令无缝集成到主程序中。
-   **模块化**: 命令逻辑与业务逻辑（Gateway/Agents）分离，便于独立测试和维护。
